21/**
22 * @brief N-buffered frame allocator.
23 * @details Maintains N frame buffers, allowing memory from the previous N-1 frames to remain valid.
24 * Useful for pipelined operations (e.g., CPU-GPU synchronization with multiple frames in flight).
25 *
26 * The allocator cycles through N buffers, ensuring that data from the previous
27 * N-1 frames remains accessible while allocating for the current frame.
28 *
29 * Uses shared mutex for read operations and exclusive lock for writes.
30 *
31 * @note Thread-safe.
32 * Previous N-1 frames' data remains valid until the buffer cycles back.
33 *
34 * @tparam N Number of frame buffers (must be > 0)
35 */
