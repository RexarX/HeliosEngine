Detailed Description

PMR arena allocator with lock-free hot allocation path.

Allocates memory from chained blocks using atomic bump pointers.

Allocation hot path is lock-free and performs one CAS on the current head block. When a block is exhausted, allocator enters a coordinated grow slow path where a single thread allocates and publishes a new block.

Individual deallocation is a no-op, matching monotonic arena semantics. Memory is reclaimed by Reset() or destruction.

Note
Thread-safe for allocate and deallocate.
Warning
Reset() must not run concurrently with allocation/deallocation.
