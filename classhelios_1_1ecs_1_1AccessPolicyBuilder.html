Detailed Description

Fluent builder for constructing an AccessPolicy.

Accumulates component and resource access declarations across any number of Query, ReadResources, and WriteResources calls. Duplicate entries are silently ignored; all sets are kept sorted.

helios::ecs::AccessPolicyBuilder()
.Query<const Transform&, const Velocity&>()
.Query<Gravity, Health*>()
.ReadResources<Camera, RenderSettings>()
.WriteResources<RenderQueue>()
.Build();

Declares a query over component types.

Component accesses are merged into the policy-wide flat read / write sets. Duplicates are silently ignored (a component already declared for writing is not re-added if the same type appears again as a write in a later Query call).

Access classification:

  • const T&, const T*, const T, T (value copy) → read
  • T&, T&&, T* → write

Tag components (empty types) are excluded from conflict tracking.
